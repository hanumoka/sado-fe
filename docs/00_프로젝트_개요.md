# SADO Frontend (sado_fe) 프로젝트 개요

> sado_be를 위한 React 프론트엔드 애플리케이션

---

## 1. 프로젝트 소개

**sado_fe**는 SADO(심장 초음파 분석 시스템) 프로젝트의 프론트엔드 애플리케이션입니다.

### 목적
- sado_be (Spring Boot 백엔드)와 연동하는 웹 프론트엔드
- 심장 초음파 DICOM 파일 업로드 및 관리 UI
- AI 분석 결과 시각화 및 리포트 조회

### SADO 시스템 아키텍처

## 2. SADO 프로젝트 구조

```
sado/
├── sado_be/     # Spring Boot 멀티모듈 백엔드 + 통합 문서 관리
├── sado_fe/     # React 프론트엔드 (현재 프로젝트)
└── sado_ai/     # Triton AI 서버 (DICOM 분석)
```

| 프로젝트 | 역할 | 기술 스택 |
|---------|------|----------|
| **sado_be** | 백엔드 API + 핵심 문서 관리 | Spring Boot 4.0.1, Java 21, Kafka, Temporal |
| **sado_fe** | 웹 프론트엔드 | React 19, TypeScript, Vite |
| **sado_ai** | AI 분석 서버 | Triton Server, gRPC, EchoNet-Dynamic |

### 문서 관리 정책
- **핵심 문서**: `sado_be/docs/` 에서 관리
- **FE 문서**: `sado_fe/docs/` 에서 관리 (현재 디렉토리)

## 5. 개발 환경 설정

### 필수 요구사항
- Node.js 18+
- npm 또는 yarn

### 설치 및 실행

```bash
# 의존성 설치
npm install

# 개발 서버 실행 (HMR)
npm run dev

# 프로덕션 빌드
npm run build

# 빌드 미리보기
npm run preview
```

### 스크립트

|---------|------|
| `npm run dev` | 개발 서버 실행 (http://localhost:5173) |

## 6. Backend 연동

### 연동 대상
sado_be의 **BFF (Backend For Frontend)** 모듈을 통해 API 연동

### 주요 API (예정)

```typescript
// Study 관리
GET    /api/bff/studies                    // 스터디 목록 조회
GET    /api/bff/studies/{studyId}/full     // 스터디 상세 + 분석 결과
POST   /api/bff/studies/{studyId}/analyze  // AI 분석 요청

// 파일 업로드
POST   /api/minipacs/dicom/upload          // DICOM 파일 업로드

// 리포트
GET    /api/bff/studies/{studyId}/report   // 분석 리포트 조회
```

### 인증 흐름

```
1. Keycloak 로그인 → JWT 토큰 발급
2. localStorage/sessionStorage에 토큰 저장
3. API 요청 시 Authorization: Bearer {token} 헤더 포함
4. Gateway에서 tenant_id 추출 → 멀티테넌시 자동 처리
```

### 설정 (예정)

```typescript
// src/config/api.ts
export const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:8080';
```
